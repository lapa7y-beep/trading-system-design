# ADR-009: 정형·비정형 데이터 교차 검증

> 상태: stable  
> 날짜: 2026-04-16  
> 결정자: jdw

---

## 문제

정형 데이터(OHLCV, 수급)와 비정형 데이터(뉴스, 공시 텍스트)가 각각 독립적으로 수집·저장되고 있으나, 두 데이터 간 교차 분석·검증 구조가 없다.

---

## 결정

정형·비정형 교차 검증은 **DP-5 (LLM → 온톨로지 조회) 경로에서 수행**한다. 실시간 매매에 개입하지 않으며, 전략 개발·백테스팅 보조 목적으로만 사용한다.

---

## 교차 검증 유형 3가지

### 유형 1 — 이벤트 기반 교차 (Event-Driven Cross)

특정 공시·뉴스 이벤트 발생 전후의 수치 데이터 변화를 분석한다.

```
비정형: DART 공시 발표일 (t0)
정형:   공시 발표 t-5일 ~ t+10일 OHLCV + 수급
분석:   공시 유형별 평균 주가 반응, 수급 변화 패턴
저장:   PostgreSQL event_impact 테이블
```

### 유형 2 — 감성·수치 상관 (Sentiment-Numeric Correlation)

뉴스 감성 점수와 기술적 지표 간 상관관계를 분석한다.

```
비정형: 뉴스 임베딩 → LLM 감성 점수 (-1.0 ~ +1.0)
정형:   동일 종목·동일 기간 RSI, 거래량 변화율, 수급
분석:   감성 점수와 수치 지표 Pearson 상관계수
저장:   TimescaleDB sentiment_correlation 하이퍼테이블
```

### 유형 3 — 시나리오 사전 검증 (Scenario Pre-Validation)

LLM이 생성한 시나리오 초안의 전제 조건을 과거 데이터로 검증한다.

```
입력:   LLM 생성 시나리오 (진입 조건, 청산 조건 포함)
검증:   TimescaleDB에서 조건 발생 횟수, 평균 수익률, MDD 조회
출력:   시나리오 신뢰도 점수 → 사람에게 제시
결정:   사람이 최종 승인 또는 수정
```

---

## 구현 위치

| 유형 | 실행 주체 | 저장소 | 실행 시점 |
|------|-----------|--------|-----------|
| 이벤트 기반 | AnalysisEngine (Python) | TimescaleDB | 배치 (일 1회) |
| 감성·수치 상관 | AnalysisEngine + LLM | TimescaleDB + pgvector | 배치 (주 1회) |
| 시나리오 사전 검증 | LangGraph Agent | TimescaleDB 조회 | 시나리오 생성 시 |

---

## 중요 경계

- 교차 검증 결과는 **참고 자료**이며 자동 매매 신호로 직결되지 않는다
- 최종 판단은 항상 사람이 한다 (사람 승인 원칙 유지)
- LLM은 교차 검증 결과 해석 보조 역할만 담당
