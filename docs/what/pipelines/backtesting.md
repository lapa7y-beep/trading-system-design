# 백테스트 파이프라인 (이중 엔진·성과지표·전환기준)

> **목적**: 벡터 기반(빠른 탐색)과 이벤트 기반(정밀 검증) 이중 엔진, MockBrokerAdapter, 성과 지표를 정의한다.
> **층**: What
> **상태**: stable
> **최종 수정**: 2026-04-16
> **구현 여정**: Step 11a(백테스트 + Sharpe 계산)에서 구현. ADR-012 §6 참조.

## 1. 핵심 원칙

**실전 코드와 최대한 공유한다.**

ClockPort·BrokerPort 두 개만 교체. StrategyRuntimePort·FSM·리스크 관리 코드는 실전과 동일.

```
settings.yaml:
  backtest:
    broker: mock   # mock | kis_paper | kiwoom_paper | kis_live | kiwoom_live
```

---

## 2. 이중 엔진

### 엔진 1 — 벡터 기반 (빠른 탐색)

```
전체 기간 데이터 → numpy/pandas 배열
지표 계산 → 신호 생성 벡터 연산
파라미터 그리드 서치 (빠른 반복)
```

용도: 파라미터 탐색, 전략 아이디어 검증

### 엔진 2 — 이벤트 기반 (정밀 검증)

```
ClockPort가 틱/봉 단위로 이벤트 발행
FSM 상태 전이 + MockBroker 체결
슬리피지·수수료·부분 체결 시뮬레이션
```

용도: 실전 배포 전 최종 검증

**흐름:** 벡터로 탐색 → 이벤트로 정밀 검증

---

## 3. MockBrokerAdapter

실전과 동일 인터페이스, 내부만 시뮬레이션.

| 항목 | 내용 |
|------|------|
| 주문 체결 | 다음 봉 시가 체결 |
| 슬리피지 | 설정 가능 (bps) |
| 수수료 | KIS/Kiwoom 실제 요율 |
| 부분 체결 | 거래량 기반 체결률 |

---

## 4. 포트폴리오 백테스트

복수 전략 × 복수 종목 동시 실행.

```
PortfolioBacktestRunner
  ├── 전략 A × 종목군 1  (독립 FSM 인스턴스)
  ├── 전략 B × 종목군 2  (독립 FSM 인스턴스)
  └── 전략 C × 종목군 3  (독립 FSM 인스턴스)

동일 종목에 복수 전략 동시 진입 시 mutex로 충돌 방지
```

## 5. 성과 지표

| 분류 | 지표 |
|------|------|
| 수익률 | CAGR, 누적 수익률 |
| 위험조정 | 샤프 비율, 소르티노 비율 |
| 낙폭 | MDD, 회복 기간 |
| 매매 통계 | 승률, 손익비 |
| 과적합 방지 | 워크포워드 구간 분할 |

결과 저장: PostgreSQL → Grafana 시각화

---

## 6. 실전 전환 기준

| 조건 | 기준값 |
|------|-------|
| 모의투자 기간 | 30일 이상 |
| 샤프 비율 | > 1.0 |
| MDD | < 15% |

기준 충족 → KIS 또는 Kiwoom 실전 전환
